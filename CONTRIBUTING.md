# Contributing to dotnet-workflow-engine

Thank you for contributing!

## Guidelines
1. **Fork & Clone**: Fork, then clone: `git clone https://github.com/your-username/dotnet-workflow-engine.git`.
2. **Branch**: Use `feature/` or `fix/` prefixes for your branch names.
3. **Build**: Ensure `dotnet build` passes.
4. **Tests**: Add unit tests for all new code. Run with `dotnet test`.
5. **Style**: Follow existing conventions, use XML doc comments, file-scoped namespaces.
6. **Commits**: Follow conventional commits: `type(scope): description`.
   - Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`.
7. **PRs**: Open a PR to `main` with a clear description and issue link.

## Issues
Use GitHub Issues to report bugs (include steps to reproduce) or request features (include problem and proposed design).
