# ExecutionContext

The `ExecutionContext` class serves as the primary state container for a specific workflow instance during its execution lifecycle within the `dotnet-workflow-engine`. It encapsulates runtime data including workflow identification, correlation tracking, temporal metrics, and scoped dictionaries for variables, activity inputs/outputs, and custom metadata. This context object allows activities to read and modify state, report completion, and handle errors while maintaining isolation between concurrent workflow executions.

## API

### Properties

#### `WorkflowInstanceId`
```csharp
public string WorkflowInstanceId { get; }
```
Gets the unique identifier assigned to the current workflow instance. This value is immutable once the context is created and is used to track the specific execution across the engine.

#### `ActivityId`
```csharp
public string? ActivityId { get; }
```
Gets the identifier of the currently executing activity within the workflow. This property is `null` when no specific activity is active (e.g., during workflow initialization or finalization).

#### `CorrelationId`
```csharp
public string CorrelationId { get; }
```
Gets the correlation identifier used to link this workflow execution with external systems, logs, or related transactions. This value is typically set upon workflow initiation.

#### `Variables`
```csharp
public Dictionary<string, object?> Variables { get; }
```
Gets the dictionary containing global variables accessible throughout the workflow instance. Modifications to this dictionary persist for the duration of the workflow and are visible to all subsequent activities.

#### `State`
```csharp
public Dictionary<string, object?> State { get; }
```
Gets the dictionary reserved for maintaining internal engine state or persistent data associated with the workflow instance. Unlike `Variables`, this is often used for system-level tracking rather than user-defined logic data.

#### `ActivityInput`
```csharp
public Dictionary<string, object?> ActivityInput { get; }
```
Gets the dictionary containing input parameters specifically mapped for the current activity. These values are populated by the engine before the activity executes.

#### `ActivityOutput`
```csharp
public Dictionary<string, object?> ActivityOutput { get; }
```
Gets the dictionary used to store output results generated by the current activity. Values added here are processed by the engine upon activity completion.

#### `StartTime`
```csharp
public DateTime StartTime { get; }
```
Gets the UTC timestamp marking the beginning of the workflow execution.

#### `EndTime`
```csharp
public DateTime? EndTime { get; }
```
Gets the UTC timestamp marking the completion of the workflow. This property is `null` while the workflow is still active.

#### `ExecutionDurationMs`
```csharp
public long ExecutionDurationMs { get; }
```
Gets the total elapsed time of the workflow execution in milliseconds. If the workflow is still running, this represents the duration from `StartTime` to the current moment.

#### `IsActive`
```csharp
public bool IsActive { get; }
```
Gets a boolean indicating whether the workflow instance is currently running. Returns `false` if the workflow has completed, faulted, or been terminated.

#### `ExecutionError`
```csharp
public string? ExecutionError { get; }
```
Gets the error message if the workflow execution failed. This property is `null` if the workflow completed successfully or is still running.

#### `Metadata`
```csharp
public Dictionary<string, object?> Metadata { get; }
```
Gets a dictionary for storing arbitrary key-value pairs associated with the execution context. This is suitable for passing contextual information that does not fit into standard variables or state.

### Methods

#### `SetVariable`
```csharp
public void SetVariable(string key, object? value)
```
Sets a global variable in the `Variables` dictionary.
*   **Parameters**:
    *   `key`: The unique name of the variable.
    *   `value`: The value to assign. Can be `null`.
*   **Throws**: Throws `ArgumentNullException` if `key` is null or empty.

#### `GetVariable`
```csharp
public object? GetVariable(string key)
```
Retrieves a global variable by name without type casting.
*   **Parameters**:
    *   `key`: The name of the variable to retrieve.
*   **Returns**: The value associated with the key, or `null` if the key does not exist or the value is explicitly null.
*   **Throws**: Throws `ArgumentNullException` if `key` is null or empty.

#### `GetVariable<T>`
```csharp
public T? GetVariable<T>(string key)
```
Retrieves and casts a global variable to the specified type.
*   **Parameters**:
    *   `key`: The name of the variable to retrieve.
*   **Returns**: The value cast to `T`, or `default(T)` if the key does not exist, the value is null, or the cast fails.
*   **Throws**: Throws `ArgumentNullException` if `key` is null or empty.

#### `SetActivityInput`
```csharp
public void SetActivityInput(string key, object? value)
```
Sets a specific input parameter for the current activity. This modifies the `ActivityInput` dictionary.
*   **Parameters**:
    *   `key`: The name of the input parameter.
    *   `value`: The value to assign.
*   **Throws**: Throws `ArgumentNullException` if `key` is null or empty. Throws `InvalidOperationException` if called when `ActivityId` is null.

#### `GetActivityInput`
```csharp
public object? GetActivityInput(string key)
```
Retrieves an input parameter for the current activity.
*   **Parameters**:
    *   `key`: The name of the input parameter.
*   **Returns**: The value associated with the key, or `null` if not found.
*   **Throws**: Throws `ArgumentNullException` if `key` is null or empty. Throws `InvalidOperationException` if called when `ActivityId` is null.

#### `SetActivityOutput`
```csharp
public void SetActivityOutput(string key, object? value)
```
Records an output result for the current activity into the `ActivityOutput` dictionary.
*   **Parameters**:
    *   `key`: The name of the output parameter.
    *   `value`: The result value.
*   **Throws**: Throws `ArgumentNullException` if `key` is null or empty. Throws `InvalidOperationException` if called when `ActivityId` is null.

#### `Complete`
```csharp
public void Complete()
```
Signals the successful completion of the current activity. This method triggers the engine to process `ActivityOutput`, update workflow state, and schedule the next activity.
*   **Throws**: Throws `InvalidOperationException` if called when `ActivityId` is null or if the context is no longer active.

## Usage

### Example 1: Reading Inputs and Setting Outputs
The following example demonstrates an activity that retrieves input values, performs a calculation, and sets the result as an output before completing.

```csharp
public class CalculateTotalActivity : IActivity
{
    public async Task ExecuteAsync(ExecutionContext context)
    {
        // Retrieve inputs specific to this activity
        var price = context.GetActivityInput<decimal>("Price");
        var quantity = context.GetActivityInput<int>("Quantity");

        // Perform logic
        var total = price * quantity;

        // Set the output for the next stage of the workflow
        context.SetActivityOutput("TotalAmount", total);
        
        // Add a global variable for reporting purposes
        context.SetVariable("LastCalculatedTotal", total);

        // Signal completion
        context.Complete();
    }
}
```

### Example 2: Managing Global State and Error Handling
This example illustrates accessing global variables, updating metadata, and handling potential execution states.

```csharp
public class UpdateStatusActivity : IActivity
{
    public async Task ExecuteAsync(ExecutionContext context)
    {
        if (!context.IsActive)
        {
            throw new InvalidOperationException("Cannot execute on an inactive workflow.");
        }

        // Read a counter from global variables
        var retryCount = context.GetVariable<int>("RetryCount");
        
        // Update metadata with execution details
        context.Metadata["ProcessedBy"] = "UpdateStatusActivity";
        context.Metadata["Timestamp"] = DateTime.UtcNow;

        // Increment and save state
        context.SetVariable("RetryCount", retryCount + 1);
        
        // Simulate conditional completion
        if (retryCount > 5)
        {
            context.SetActivityOutput("Status", "Failed_MaxRetries");
        }
        else
        {
            context.SetActivityOutput("Status", "Processing");
        }

        context.Complete();
    }
}
```

## Notes

*   **Thread Safety**: The `ExecutionContext` is not thread-safe. In scenarios where an activity spawns background tasks or uses asynchronous continuations that might access the context concurrently, external synchronization (e.g., `lock` statements) is required when accessing mutable members like `Variables`, `State`, `ActivityInput`, `ActivityOutput`, and `Metadata`.
*   **Activity Scope**: Methods prefixed with `Activity` (`SetActivityInput`, `GetActivityInput`, `SetActivityOutput`, `Complete`) rely on the presence of a valid `ActivityId`. Invoking these methods when `ActivityId` is `null` (such as in workflow-level event handlers) will result in an `InvalidOperationException`.
*   **Null Handling**: The generic `GetVariable<T>` method returns `default(T)` rather than throwing if a key is missing or the type cast fails. Callers must validate the return value if the variable is expected to exist.
*   **Immutability**: Identification properties (`WorkflowInstanceId`, `CorrelationId`, `StartTime`) and status flags (`IsActive`, `ExecutionError`) are read-only. Attempting to modify these directly is not supported; state changes for these fields are managed internally by the workflow engine.
*   **Completion Semantics**: Calling `Complete()` does not immediately terminate the entire workflow; it only finalizes the current activity. The workflow instance remains `IsActive` until the final activity completes or a terminal error occurs.
