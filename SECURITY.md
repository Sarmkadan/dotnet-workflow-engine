# Security Policy

## Reporting a Vulnerability

We take the security of dotnet-workflow-engine seriously. If you discover a security vulnerability, we appreciate your help in disclosing it to us in a responsible manner.

Please do not report security vulnerabilities through public GitHub issues.

Instead, please report them by:
- Using GitHub's built-in private advisory feature.
- Or contacting us directly via email at rutova2@gmail.com.

### Response Time
We will strive to acknowledge receipt of your vulnerability report within 48 hours and provide regular updates on the resolution process.

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |
